11计算机网络1 班-任锦

工作方向 Linux运维/自动化运维/运维开发
毕业学校 平遥现代工程技术学院11网络1班
毕业学校 天津滨海职业学院14电信2班
个人技术网站 https://www.asjin.com

为何先择这所学校\班级

  • 从初中毕业那会，对文化课学习不是强项, 中考成绩不是很好，当时家里对我不抱有希望。但是我对自己比较了解，当时看重了计算机专业的前景，并且对计算机专业有浓厚的兴趣爱好，立志学习计算机网络专业技术。从2011年那时起，就决定先改变自己，而后带动家庭，在家庭中做为一个榜样，最后在家人的鼓励下为社会做贡献，有了三大步发展…

火车跑的快，全靠车头带

  • 当时刚上到我们班级时，我属于成绩平平，技术占底的学生。并且说到底，我也很笨。但是当时遇到我们好的雷老师，是我们专业学习的领头人和兴趣爱好激发者，这点毋庸置疑；我的技术文化课在持续的一步步的上升~
  • 从最基本的五笔打字一分钟7个字，到现在的一分钟100个字，快速且精准的后台服务操作; 从最基本的计算机应用基础，到现在的工作自动化流程化，持续集成; 从最基础的网络基础到现在的高可用部署与应用体系的架构部署、实现、切分、优化; 从最基础的计算机原理，到系统的快速监控、告警、通知、故障处理与自动修复; 从基础的做人、做事、孝亲、尊师到对家庭发展的带动，对职业的爱好，对岗位的敬重，无一离不开老师对我们做人做事，专业文化的谆谆教诲。

对专业领域的抉择

  • 计算机网络领域其实很大，涉及的专业和领域也都很广。记得在高二末，老师带我们做vmware虚拟化windows server 2003 ，搭建的的服务时，激发了我对系统对应服务体系的强大爱好。并且记得老师在讲计算机操作系统分类时，讲到过”Linux 系统的功能很强大，应用领域也非常广，大家下来还要好好研究一下”。从那时起，我便决定发展linux系统与服务体系方向。可以说学习方面是着实的笨，但是贵在有一个领头人，好在有一个持续的学习状态。把简单的东西做到极致是难能可贵的，强大的计算机不就是”0”,”1” 两个数字做到了极致么？

选择的坚持与发展

  • 9年计算机网络相关专业的学习与坚持;
  • 6年linux使用经验;
  • 4年+ LANMP 项目运维经验，数据库维护经验;
  • 3年kvm、docker、kubernetes私有云建设、维护经验;
  • 喜欢编程、工作细心认真、责任心强、善于从多角度的分析问题、解决问题;
  • 热爱开源技术、支持运维自动化、避免重复性工作、工作成果文档化、邮件日志规范化。

业务技能

  1. 熟悉Linux 系统管理基础知识及原理、熟练各种常用运维命令的操作;
  2. 熟悉Iptables、SSH、DNS、DHCP、VSFTP、Supervisor、OpenVPN等基础服务的配置;
  3. 熟悉Zabbix、Prometheus、Grafana 、ELK等监控展示工具的使用;
  4. 熟悉Ansible、Fabric、Git、Jenkins等自动化工具编写与维护;
  5. 熟悉KVM、Docker、kubernetes 等项目相关原理、基本架构及简单实现;
  6. 熟悉Apache、Nginx、Tomcat等web中间件的日常变更部署及故障处理;
  7. 熟悉LVS、Keepalived、Nginx、Haproxy 等高可用负载均衡项目部署;
  8. 熟悉MySQL、Redis、MongoDB、Memcached 等数据库项目部署;
  9. 熟悉shell、python 编程语言，能够对应公司需求编写自动化工具;
  10. 熟悉sge 高性能集群、分布式存储部署与维护;

工作经历

2020 至今 天津诺禾致源生物信息科技有限公司 规模:3000人以上

  1. zabbix + grafana 自动化监控构建与维护;
  2. gitlab + keepalived + lsyncd 高可用服务构建与维护;
  3. sge 高性能集群自动化部署，分布式布存储构建与维护;
  4. P级数据扫描: awk + bash + python 实现自动化分析与汇总;
  5. 全公司各级部门计算资源使用量，实时监控生成数据比例，自动化分摊项目开发(python+bash+awk);
  6. 各部门下机基因测序数据生成，P级超期数据切割分离、各业务线下机超期数据自动化公示。下机数据推进删除;
  7. 日常oa处理: 集群下机数据删除，集群用户权限变更，集群节点调整，用户新建等。

2017-07 至 2020 北京华图宏阳教育文化发展股份有限公司 总部 Linux 运维工程师 规模1w人以上

  1. 负责kvm 、docker/kubernetes 私有云资源日常开通、建设、部署、规划与维护等工作;
  2. 负责ansible、python/fabric自动化工具、脚本编写与维护，以降低维护成本，提高运维效率;
  3. 负责公司应用系统的日常运维: 包括应用配置管理、故障处理、系统监控、系统升级、系统迁移等工作;
  4. 研究应用系统服务架构，提高应用系统的稳定性及高可用性;
  5. 配合公司各大部门、技术中心、研发中心等日常运维支持;
  6. 公司内部基础服务搭建、线上服务可用性、稳定性保障;
  7. 系统故障快速报警和处理，服务器安全加固;
  8. 根据产品线不同，编写对应条产品线的应急预案;
  9. 为提升工作效率，编写python自动化工具;

项目经历

2017-08 至 2017-11 项目构架切分升级 智能DNS + Nginx 高可用负载均衡集群部署

项目描述

因公司是互联网教育公司，海外用户访问量剧增，而当时我们的生产环境存在单点失败的风险，经过公司领导讨论决定对项目架构按生产线进行切分与升级。

架构切分

针对公司环境采用LNMP 架构，用智能DNS将海外各国请求，解析到离这个国家最近的Nginx负载均衡器，由Nginx将以上发来的请求按session绑定在国外集群中进行调度，最终将请求发往海外集群中的服务器进行响应，MySQL为主从配置，新的架构足以支撑日均pv 200W以上。

项目职责

  1. 服务器的基础环境建设、SSH认证及安全策略的配置。
  2. 智能DNS + Nginx+LNMPP+ Lua + Redis + ELK + Prometheus 高可用负载均衡集群搭建。
  3. 对新服务进行各方面的必要测试。
  4. 配合测试、开发等相关人员进行沟通协调，项目文档的完善。

2017-03 至 2017-05 线上服务器全面监控系统的部署配置

项目描述

针对公司架构行进切分升级、云平台各服务进行重新部署优化，需要对线上的系统、服务及相关行业务指标进行监控;项目采用Zabbix监控及Python、Shell 三套应用进行监控。

项目职责

  1. 编写Shell监控脚本，将脚本部署在公司内网环境中，用于对阿里云平台中的所有主机的网络进行监控，确保所有使用所阿里云产品的网络是可用的。
  2. 将安装部署Zabbix基础服务到阿里云ECS上，配合shell脚本对监控进行配置、调整、将报警信息通过微信、短信、企业邮箱报警到相关负责人，并添加告警升级。
  3. 因客户是有大部分是海外的，配合爬虫团队编写爬国际ip代理监控脚本，确保从业务方可以正常访问三条生产线的open接口。

2017-06 至 2017-08 架构优化与迁移

项目描述

针对海外收单的需求，B/S 架构的平台响应一直不是很理想，并且阿里海外服务器费用相对较高，直接部署代码到海外很费时。因此考虑将应用迁回到国内服务器。弃用阿里云海外结点，而使用青云服务器集群做中转。

项目职责

  1. 部署nginx调度器到青云服务器。
  2. 智能DNS变更到青云服务器中转组。
  3. 优化后从海外国家响应速度为200ms。

2018-01 至 2018-03 自动化运维脚本开发与ELK部署

项目描述

针对所负责的运维项目，用ansible只能实现半自动化运维，后依据公司三条不同的生产线，用python + shell 编写自动化脚本、并且完善、用于系统初始化配置、项目更新发布、数据备份恢复。

针对开发查看生产日志的需求，部署ELK 到内网环境，将特定的端口进行映射与反代，将nginx 日志、web系统日志、python系统日志用filebeat 发往内网服务器的logstash。

2018-09 至 2018-10 keepalived + nginx+rsync 服务双主高可用部署

项目描述

针对各大项目高可用需求，使用两台物理服务器，用ansible推送keepalived + nginx + rsync 双主高可用应用，使得物理服务器故障时可以完成秒级切换。提升公司各大项目应用高可用性。

2018-11 至 2019-02 kvm 自动化开通服务器工具开发

针对日常开通kvm私有云服务器需求，结合libguestfs-tools-c工具突破公司原kvm自动化工具不能自动分配虚拟机ip问题，完成更高效率的开通提升。

  • 每台服务器创建效率比以往提升69%；

2019-04 至 2019-07 docker+ kubernetes 私有云集群建设

  • 首先将测试环境与beta环境建设Docker+ Kubernetes 私有云，整合Jenkins Gitlab 完成Kubernetes 初步测试与自动化构建运行，后逐步将各项Kvm生产项目打包构建至Kubernetes私有云。
